# ADR-004: عمود حدثيّ بين السياقات (مُصفوف/idempotent/مُهدَّأ)

## القرار
التكامل **بين** Bounded Contexts يتمّ حصراً عبر Domain Events — مُصفوفة
(versioned)، idempotent، وقابلة لإعادة المحاولة (retriable). المستهلكون
الثقال (مثل إعادة بناء الصفحة الرئيسيّة) مُهدَّؤون/مُجمَّعون (debounced/
batched)، لا يُنفَّذون تزامنيًّا لكلّ حدث فرديّ.

## السبب
فكّ الاقتران بين السياقات، وموثوقيّة التسليم، وقابليّة إضافة مستهلكين
جدد (AI، تحليلات، فهرسة بحث) دون تعديل مسار النشر نفسه في كلّ مرّة.

## البدائل المرفوضة
- **fan-out أمريّ مبعثر** — نداءات مباشرة من سياق لآخر منتشرة عبر الكود؛
  كلّ مستهلك جديد يعني تعديل نقطة النشر نفسها، ويُنتج اقتراناً كثيفاً
  يصعب اختباره وتتبّعه.
- **نداءات متزامنة عابرة للسياقات** — تُبطئ مسار الكتابة بمقدار أبطأ
  مستهلك، وتُسقِط الطلب كلّه عند فشل أيّ مستهلك واحد.

## الأثر
إضافة أثر جانبيّ جديد = إضافة listener جديد، بلا لمس مسار النشر
الأصليّ؛ كلّ سياق يستهلك الأحداث التي تعنيه فقط، بمعزل عن تفاصيل تنفيذ
السياقات الأخرى.
