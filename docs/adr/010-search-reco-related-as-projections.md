# ADR-010: Search/Reco/Related projections في سياقاتها، تُستعلَم عبر APIها

## القرار
البحث (Search)، التوصية (Recommendation)، والمحتوى ذو الصلة (Related)
كلّها **Projections** تعيش داخل سياقاتها الخاصّة (Search وIntelligence)،
تُستعلَم فقط عبر API السياق (`SearchAPI`, `RecommendationAPI`) — لا عبر
استعلام مباشر لجداول الكتابة.

## السبب
أداء (فهرسة/بحث متخصّص أسرع بكثير من استعلام SQL نصّيّ) وفصل واضح بين
مصدر الحقيقة وآليّة الاستكشاف/التوصية.

## البدائل المرفوضة
- **بحث DB LIKE** (استعلام `LIKE`/`WHERE` نصّيّ مباشر على جداول الكتابة)
  — أداء ضعيف عند الحجم، ولا يدعم صلة (relevance) حقيقيّة أو بحثاً
  متعدّد اللغات/الحقول بكفاءة.
- **ربط القراءة بالكتابة** (استعلام مباشر لجداول المحتوى لبناء قوائم
  التوصية/ذات الصلة عند كلّ طلب) — يُبطئ القراءة ويربط شكل البيانات
  المطلوب للتوصية ببنية جداول الكتابة.

## الأثر
استبدال محرّك البحث أو خوارزميّة التوصية لاحقاً يحدث **خلف الحدّ**
(SearchAPI/RecommendationAPI) دون أن يلمس أيّ مستهلك لهذه الواجهات.
