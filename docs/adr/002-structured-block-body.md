# ADR-002: جسمٌ كتليّ مُنظَّم؛ `content_html` ليس مصدر حقيقة

## القرار
جسم المحتوى مُخزَّن ككتل مُنظَّمة (`ContentBlock[]`)، لا كسلسلة HTML واحدة.
`content_html` (إن وُجد) مُشتقّ للعرض فقط — أبدًا مصدر الحقيقة.

## السبب
يخدم توزيعاً عبر قنوات متعدّدة (omnichannel — ويب/موبايل/AMP/شركاء) دون
تحليل HTML؛ يجعل المحتوى قابلاً للاستعلام والتحويل الآليّ (AI: تلخيص،
إعادة كتابة، استخراج كيانات لكلّ كتلة على حِدة)؛ ويسمح بإعادة عرض مختلفة
لكلّ قناة من نفس المصدر البنيويّ.

## البدائل المرفوضة
- **HTML أحاديّ** (حقل نصّيّ واحد لكامل الجسم) — أبسط تنفيذاً، لكنّه يمنع
  أيّ معالجة بنيويّة (AI لكلّ كتلة، عرض مختلف لكلّ قناة، بحث لكلّ كتلة)
  دون parsing هشّ لاحقاً.

## الأثر
محتوى قابل للاستعلام والتحويل بنيويًّا؛ إعادة العرض لكلّ قناة (ويب مقابل
موبايل مقابل AMP) لا تتطلّب parsing HTML، بل قراءة مباشرة لبنية الكتل.
