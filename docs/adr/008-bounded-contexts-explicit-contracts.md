# ADR-008: سياقات محدودة بعقود صريحة؛ لا وصول قاعدة عابر

## القرار
المنصّة مقسَّمة إلى Bounded Contexts صريحة (Identity, Content, Editorial,
Media, Search, Intelligence/AI, Advertising, Notification, Analytics,
Sports, Live, Delivery). التكامل **داخل** السياق نداءات مباشرة؛ **بين**
السياقات: أحداث + Public API فقط. **لا** وصول مباشر لقاعدة بيانات/جداول
سياقٍ آخر (لا joins عابرة للسياقات).

## السبب
استقلاليّة كلّ سياق في التطوّر والاختبار والنشر، وقابليّة استخراج أيّ
سياق لاحقاً كخدمة منفصلة (راجع ADR-014) دون الحاجة لإعادة تصميم عقود
التكامل — لأنّها مصمَّمة كعقود صريحة من اليوم، لا كوصول مباشر يصعب فكّه
لاحقاً.

## البدائل المرفوضة
- **مخطّط قاعدة بيانات مشترك بلا حدود** — يُسهِّل التطوير قصير المدى لكن
  يُنتج اقتراناً خفيًّا كثيفاً (أيّ سياق يمكنه قراءة/كتابة أيّ جدول)
  يصعب فكّه لاحقاً عند الحاجة لاستخراج خدمة.
- **joins عابرة للسياقات** — تُبقي أداء الاستعلام مرتبطاً ببنية جداول
  سياقات أخرى قد تتغيّر بمعزل عنك.

## الأثر
كلّ سياق يتطوّر ويُختبَر ويُستخرَج (إلى خدمة مستقلّة، عند الحاجة) بمعزل
عن تفاصيل تنفيذ السياقات الأخرى، طالما العقد الصريح (API/أحداث) لم يتغيّر.
