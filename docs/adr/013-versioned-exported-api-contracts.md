# ADR-013: عقود API مُصدَّرة ومُصدَرة بإصدارات (ComposedDocument/Content) لرؤوس متعدّدة

## القرار
عقود الـAPI (شكل `ComposedDocument`، شكل `Content` المُصدَّر) رسميّة
ومُصدرة بإصدارات (versioned) — مصمَّمة لخدمة رؤوس (frontends) متعدّدة:
الويب، تطبيقات الموبايل، وشركاء خارجيّين.

## السبب
تطبيقات الموبايل والشركاء الخارجيّين يعتمدون على استقرار شكل الـAPI؛
تغييرات غير مُصدَرة بإصدار تكسر عملاء منشورين لا سيطرة مباشرة على توقيت
تحديثهم (خاصّة تطبيقات الموبايل، حيث التحديث يعتمد على قبول المستخدم
والمتجر).

## البدائل المرفوضة
- **عقود بلا إصدار** (شكل API يتغيّر بحرّيّة مع كلّ تطوّر داخليّ) — يكسر
  أيّ عميل خارجيّ (تطبيق موبايل منشور، شريك API) عند أوّل تغيير غير
  متوقَّع.

## الأثر
لا كسر للتطبيقات المنشورة عند تطوّر النموذج الداخليّ؛ التطوّر الداخليّ
(مثل الانتقال إلى `ContentItem`) يحدث خلف عقد ثابت، مع إصدارات جديدة من
العقد تُضاف صراحة عند الحاجة لتغيير غير متوافق خلفيًّا.
