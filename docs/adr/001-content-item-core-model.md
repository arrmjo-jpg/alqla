# ADR-001: النواة `ContentItem` عامّة (Arc/Ghost هجين)

## القرار
نواة المنصّة عقدة محتوى عامّة واحدة، `ContentItem` — غلاف مُطبَّع (دورة
حياة/تصنيف/أعلام/مراجعات/cache-tags/بحث) موحَّد لكلّ الأنواع، مع حمولات
مُنمَّطة ذات رأي لكلّ نوع (article/video/podcast/reel/…) وجسمٌ كتليّ
(`ContentBlock[]`). نمط **Arc/Ghost الهجين**: غلاف عامّ + حمولة نوعيّة +
كتل — لا EAV، ولا genericity بلا رأي.

## السبب
يوحّد الغلاف لكلّ الأنواع **مرّة واحدة**؛ شرطٌ مسبق للـComposer (الـPins
تشير لعقدة عامّة، لا لنوع بعينه) وللتوصية والبحث عبر الأنواع (query واحد
عبر كلّ المحتوى بدل UNION عبر جداول منفصلة).

## البدائل المرفوضة
- **صوامع Article-only** — كلّ نوع محتوى جدولٌ وكودٌ منفصل بالكامل؛ يمنع
  أيّ ميزة عابرة للأنواع (بحث موحَّد، تثبيت موحَّد، توصية موحَّدة) دون
  تكرار الجهد لكلّ نوع.
- **EAV meta** (فخّ WordPress) — مرونة زائفة بتكلفة استعلام باهظة وفقدان
  الأنواع القويّة.
- **genericity بلا رأي** (فخّ Contentful) — مخطّط «يبني أيّ شيء» يفقد
  المعرفة النطاقيّة اللازمة لقواعد نشر/تحقّق محدَّدة النوع.
- **Field API** (نمط Drupal) — تعقيد تكوين زائد لحاجة لا تستدعيه بنية
  الأنواع الحاليّة والمخطَّطة.

## الأثر
كلّ اهتمامات الغلاف (دورة الحياة، الأعلام، الـcache tags، البحث) تُكتب
**مرّة واحدة** بدل تكرارها لكلّ نوع؛ الأنواع الجديدة تصبح شبه-مجّانيّة
(حمولة + قدرات، بلا إعادة بناء الغلاف).
